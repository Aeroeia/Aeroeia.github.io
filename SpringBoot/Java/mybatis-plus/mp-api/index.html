BaseMapper、IService方法使用

Aer Lv3
  2025-04-19 15:38:13 2025-04-19 15:38:13 Created   2025-04-28 19:26:04 2025-04-28 19:26:04 Updated  
  • SpringBoot

在 MyBatis-Plus（简称 MP）中，IService 是一个通用的 Service 接口，封装了对单一实体类的常用数据库操作。它是 MyBatis-Plus 提供的一个顶层接口，继承自 IService 的实现类（比如 ServiceImpl）可以直接调用这些方法，避免了写大量的重复代码。
下面是 IService 接口中主要的方法以及详细介绍：

一、基本的 CRUD 操作

方法 说明
boolean save(T entity) 插入一条记录。
boolean saveBatch(Collection entityList) 批量插入记录（默认每次插入 1000 条）。
boolean saveBatch(Collection entityList, int batchSize) 指定批次数量的批量插入。

二、删除操作

方法 说明
boolean removeById(Serializable id) 根据 ID 删除。
boolean removeByMap(Map<String, Object> columnMap) 根据 columnMap 条件删除记录。
boolean remove(QueryWrapper queryWrapper) 根据条件构造器删除记录。
boolean removeByIds(Collection<? extends Serializable> idList) 批量根据 ID 删除。

三、更新操作

方法 说明
boolean updateById(T entity) 根据 ID 更新记录。
boolean update(T entity, UpdateWrapper updateWrapper) 根据条件构造器更新记录。
boolean updateBatchById(Collection entityList) 批量更新记录（默认 1000 条）。
boolean updateBatchById(Collection entityList, int batchSize) 指定批次数量的批量更新。

四、查询操作

方法 说明
T getById(Serializable id) 根据 ID 查询记录。
List listByIds(Collection<? extends Serializable> idList) 根据多个 ID 查询记录。
List listByMap(Map<String, Object> columnMap) 根据 columnMap 查询记录。
T getOne(QueryWrapper queryWrapper) 获取一条记录，若有多条会抛异常。
T getOne(QueryWrapper queryWrapper, boolean throwEx) 同上，throwEx 为 false 则返回第一条，不抛异常。
Map<String, Object> getMap(QueryWrapper queryWrapper) 获取一条记录并转为 Map。
List<Map<String, Object>> listMaps(QueryWrapper queryWrapper) 查询多条记录，返回 Map 列表。
List<Object> listObjs(QueryWrapperqueryWrapper) 查询多条记录，仅返回某个字段的值（通常用于单列查询）。
long count() 查询总记录数。
long count(QueryWrapper queryWrapper) 条件统计记录数。

五、分页操作

方法 说明
IPage page(IPage page) 无条件分页查询。
IPage page(IPage page, QueryWrapper queryWrapper) 条件分页查询。
IPage<Map<String, Object>> pageMaps(IPage page, QueryWrapper queryWrapper) 条件分页查询，返回 Map。

六、列表查询

方法 说明
List list() 查询所有记录。
List list(QueryWrapper queryWrapper) 条件查询记录列表。

七、保存或更新

方法 说明
boolean saveOrUpdate(T entity) 根据 ID 判断是插入还是更新。
boolean saveOrUpdateBatch(Collection entityList) 批量插入或更新。
boolean saveOrUpdateBatch(Collection entityList, int batchSize) 批量插入或更新并指定批次数。

八、常见泛型类型

  • T：实体类的类型。
  • IPage：分页接口，常用实现类是 Page。
  • QueryWrapper：用于构造查询条件的工具类。
  • UpdateWrapper：用于构造更新条件的工具类。

MyBatis-Plus 的 BaseMapper 是对 MyBatis 的 Mapper 层接口 的封装，它为每个实体类提供了基础的 CRUD 操作，免写 XML 和接口方法，大大提高了开发效率。

✅ 一、什么是 BaseMapper

BaseMapper 是 MyBatis-Plus 提供的基础接口，它封装了大量通用的数据库操作方法。你只需要写一个接口继承它，就能自动拥有各种 CRUD 能力。

✅ 二、BaseMapper 的常用方法详解

方法 说明
int insert(T entity) 插入一条记录。成功返回影响行数。
int deleteById(Serializable id) 根据主键 ID 删除。
int deleteByMap(Map<String, Object> columnMap) 根据 columnMap 条件删除记录。
int delete(QueryWrapper wrapper) 根据条件构造器删除。
int deleteBatchIds(Collection<? extends Serializable> idList) 根据多个 ID 批量删除。
int updateById(T entity) 根据主键 ID 更新记录。
int update(@Param(Constants.ENTITY) T entity, @Param(Constants.WRAPPER) Wrapper updateWrapper) 根据条件构造器更新记录。
T selectById(Serializable id) 根据主键 ID 查询记录。
List selectBatchIds(Collection<? extends Serializable> idList) 批量 ID 查询记录。
List selectByMap(Map<String, Object> columnMap) 根据 columnMap 查询记录。
T selectOne(QueryWrapper wrapper) 查询一条记录，如果结果多于一条会报错。
Integer selectCount(QueryWrapper wrapper) 根据条件查询记录数。
List selectList(QueryWrapper wrapper) 根据条件查询列表。
List<Map<String, Object>> selectMaps(QueryWrapper wrapper) 查询列表并返回 Map 集合。
List 查询某一列的所有值集合。
IPage selectPage(IPage page, QueryWrapper wrapper) 分页查询记录。
IPage<Map<String, Object>> selectMapsPage(IPage page, QueryWrapper wrapper) 分页查询 Map 类型数据。

✅ 四、使用小贴士

  1. BaseMapper 是 MyBatis-Plus 的核心，封装的是最基本的数据库操作，适合简单直接的增删改查操作。
  2. 如果逻辑比较复杂，建议在 Service 层通过组合方式来处理，而不是在 Mapper 层拼很多复杂语句。
  3. @Mapper 注解必须加（或者在启动类用 @MapperScan 扫描路径）。
  4. 不支持批量插入（insertBatch），如果需要请在 XML 中自定义。
