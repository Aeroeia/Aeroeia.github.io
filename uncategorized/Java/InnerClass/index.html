一文讲清Java内部类：全面解析静态与非静态内部类

​​一、成员内部类​​（非静态内部类）

成员内部类属于外部实例，必须创建外部实例才能创建成员内部类实例

基本用法

特点

  • 成员内部类可以访问外部类的所有成员（包括private和static）
  • 成员内部类属于外部实例,一个外部类实例可以有多个内部类实例
  • 必须先创建外部类实例才能创建内部类实例,
  • 内部类中可以使用Outer.this显式引用外部类实例

java16以前成员内部类不允许定义静态成员/静态方法

二、静态嵌套类（静态内部类）

​​静态内部类（static inner class）不属于外部类的实例​​，它更像是一个​​独立​​的类，只是​​嵌套在另一个类的作用域内​​。可以把它理解为一个普通的类，只是它的类名是 Outer.StaticInner，而不是 StaticInner，这也解释了为什么静态内部类只能调用外部类的静态成员

基本用法

成员内部类 vs 静态内部类 对比

特性 成员内部类（非静态内部类） 静态内部类
依赖外部类实例 ✅ 必须依赖（outer.new Inner()） ❌ 不依赖（直接 new Outer.Inner()）
访问外部类实例成员 ✅ 能访问（包括 private 成员） ❌ 不能访问（只能访问 static 成员）
访问外部类静态成员 ✅ 能访问 ✅ 能访问
定义 static 成员 java16以前不能（除非 static final 常量）java16后可以 ✅ 可以定义 static 方法/字段
内存占用 隐式持有外部类引用（this$0） 不持有外部类引用，更独立
典型用途 需要访问外部类实例的场景（如事件监听） 工具类、Builder 模式、线程安全单例
创建方式 Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();		 Outer.Inner inner = new Outer.Inner();

关键区别总结

  1. 依赖关系

    • 成员内部类必须绑定到外部类实例（outer.new Inner()）。
    • 静态内部类独立存在（直接 new Outer.Inner()）。

  2. 访问权限

    • 成员内部类可以访问外部类的所有成员（包括 private 实例字段/方法）。
    • 静态内部类只能访问外部类的 static 成员。

  3. 静态成员支持

    • 成员内部类不能有 static 方法/字段（除非是 static final 常量）。
    • 静态内部类可以自由定义 static 成员。

  4. 内存管理

    • 成员内部类隐式持有外部类引用（可能导致内存泄漏）。
    • 静态内部类不持有外部类引用，更安全。
