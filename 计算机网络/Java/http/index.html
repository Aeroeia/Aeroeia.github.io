总结对比

Content-Type 后端接收方式 前端发送方式 典型场景
application/json @RequestBody JSON.stringify(data) REST API 数据交互
multipart/form-data @RequestParam + MultipartFile FormData 对象 文件上传
application/x-www-form-urlencoded @RequestParam URLSearchParams 或 key=value&… HTML表单提交

关键注意事项

  1. multipart/form-data
    • 前端不能手动设置 Content-Type，浏览器会自动添加 boundary。
    • 后端用 MultipartFile 接收文件。

  2. application/json
    • 前端需手动调用 JSON.stringify() 转换数据。
    • 后端用 @RequestBody 绑定到对象。

  3. x-www-form-urlencoded
    • 数据格式为 key=value&key2=value2（需URL编码）。
    • 后端用 @RequestParam 接收字段。
